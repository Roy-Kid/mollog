## What this repo is

`mollog` is a structured logging library for the molcrafts ecosystem
(Python 3.12+, runtime dependency-free except for `rich`). It ships
trace/debug/info/warning/error/critical levels, structured `extra=`
fields, JSON / stdlib-style / rich formatters, a bridge that funnels
stdlib `logging` into mollog, and an optional Pydantic Logfire
integration. Distributed on PyPI as `molcrafts-mollog`.

## What must never change casually

- The top-level public API in `src/mollog/__init__.py` — the
  stdlib-flavored helpers (`trace`, `debug`, `info`, `warning`,
  `error`, `critical`, `exception`), `get_logger`, `configure`,
  `set_level`, the level constants, and `LogRecord`. Per
  `CONTRIBUTING.md`, public APIs follow semver — breakages are
  deliberate and scheduled for major releases only.
- The `StdlibBridgeHandler` contract in `_stdlib_bridge.py` —
  third-party libraries route through this handler; behavior
  changes propagate widely.
- The Pydantic Logfire integration shape in `_logfire.py` — depends
  on Logfire's public API; pin and verify on dependency bumps.
- The runtime package stays dependency-free apart from `rich`
  (per CONTRIBUTING.md change expectations).
